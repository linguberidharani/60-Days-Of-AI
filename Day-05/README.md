# Day 5 - Context Engineering

## Objective

Learn how providing context improves AI-generated outputs.

## Prompt A (Without Context)

Prompt:

Create a 30-day learning roadmap.

Include:

* Weekly milestones
* Daily tasks
* Resources
* Projects
* Final outcome

Make it practical and beginner-friendly.

### Observation

Claude first needed clarification about the topic and then generated a generic roadmap for Prompt Engineering and AI tools.

The roadmap was useful but not personalized to my goals.

---

## Prompt B (With Context)

Prompt:

Create a 30-day learning roadmap.

Context:

* Current Situation: Student
* Current Skills: Cyber Security Fundamentals
* Goal: To learn Cyber Security tools
* Available Time: 1 Hour per Day
* Experience Level: Beginner
* Preferred Learning Style: Videos & Projects & Reading

### Observation

Claude generated a highly personalized cybersecurity roadmap focused on:

* Kali Linux
* Nmap
* Wireshark
* Burp Suite
* Metasploit
* John the Ripper
* Hashcat
* Splunk

The roadmap also respected my available study time and preferred learning style.

---

## Comparison

### Which roadmap feels more personalized?

Prompt B because it was customized based on my goals, skills, experience level, and available time.

### Which roadmap would I actually follow?

Prompt B because it provides a practical path toward learning cybersecurity tools and building hands-on experience.

### What role did context play?

Context helped Claude understand my background and goals, allowing it to create a roadmap that was realistic, relevant, and actionable.

---

## Key Learnings

1. Context dramatically improves AI outputs.
2. The more relevant information you provide, the better the response becomes.
3. Personalized prompts lead to more actionable recommendations.
4. Context Engineering is one of the most powerful prompt engineering techniques.

## Biggest Takeaway

AI performs best when it understands the user's situation. Context transforms generic answers into personalized solutions.
